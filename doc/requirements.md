## 需求详情

### 需求001：单词点击翻译

**功能描述：**
在双语歌词区，点击任意英文单词，弹出翻译词典风格的弹窗。

**弹窗内容：**
1. 单词本身
2. 音标
3. 发音：英语发音 + 美语发音（可点击播放）
4. 词性（noun, verb, adj. 等）
5. 释义（中文解释）
6. 3个例句

**弹窗交互：**
- 弹窗显示在屏幕正中央，与单词位置无关
- 点击关闭按钮（×）可关闭弹窗
- 点击弹窗外部区域可关闭弹窗

**歌词区域交互调整：**
- 每行歌词右侧增加一个独立的播放按钮（喇叭图标）
- 点击播放按钮 → 播放该句音频
- 点击单词 → 弹出翻译弹窗
- 两者独立，不冲突

**技术实现：**
- 调用 Google Translate API 获取翻译结果
- 单词用 `<span>` 包裹，绑定点击事件
- 弹窗宽度约 320px

**样式要求：**
- 与页面现有样式保持一致（圆角、阴影等）

## 需求详情

### 需求002：PDF教材查看功能

**功能描述：**
在页面右侧添加 PDF 教材预览区域，用户可查看对应课本的 PDF 内容。

**布局调整：**
```
┌─────────────────────────────────────────────────┐
│  Header (全宽) - 不变                            │
├─────────────────────────────────────────────────┤
│  Book info + Course selection (全宽) - 不变       │
├───────────────────────────┬─────────────────────┤
│  Player controls          │                     │
│  (宽度50%)               │   PDF Viewer        │
├───────────────────────────┤   (宽度50%)        │
│  Lyrics/Content area     │                     │
│  (宽度50%)               │                     │
└───────────────────────────┴─────────────────────┘
```

**技术实现：**
- 使用 pdf.js 库渲染 PDF
- PDF 文件放在 `pdf/` 目录作为静态资源
- 用户选择不同课本时，切换加载对应的 PDF 文件

**交互说明：**
- PDF 区域可滚动，用户自行翻页查找内容
- 不做页码与课程的自动对应，由用户自行翻阅
